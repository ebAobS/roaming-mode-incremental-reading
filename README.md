        <h2 style="color: #be185d; margin: 0 0 1em; text-align: center; font-size: 1.3em;">💡 Core Philosophy</h2>
        <div style="background: white; padding: 1.5em; border-radius: 8px; border-left: 4px solid #ec4899;">
            <p style="margin: 0.5em 0; line-height: 1.6;"><strong>Incremental reading should be distinguished from flashcards:</strong></p>
            <ul style="margin: 0.5em 0; padding-left: 1.2em;">
                <li style="margin: 0.3em 0;"><strong>Flashcards</strong> review algorithm principle is <strong>forgetting curve</strong>, for which SiYuan flashcard system + FSRS algorithm is already sufficiently perfect. Years of experience using Anki and SuperMemo tell us that the "minimum information principle" for cards is extremely important.</li>
                <li style="margin: 0.3em 0;"><strong>Incremental reading</strong> deals with large text segments, inherently not conforming to the "minimum information principle". Most current incremental reading solutions habitually use forgetting curve-based review algorithms (like FSRS) to recommend article reviews, which is not reasonable.</li>
            </ul>
            <p style="margin: 0.5em 0; line-height: 1.6;">For a lengthy article, it's difficult to <strong>solely use memory level</strong> as the standard for whether to <strong>be recommended for review next time</strong>. The real <strong>standard should be multi-dimensional</strong>, such as difficulty, learning progress, importance of learning content, urgency, interest level, etc.</p>
            <p style="margin: 0.5em 0; line-height: 1.6;"><strong>In summary, the core purpose of incremental reading is not to "fight forgetting", but to achieve stress-free "read later", ultimately efficiently learning large amounts of material simultaneously.</strong></p>
